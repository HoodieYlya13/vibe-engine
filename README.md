# Vibe Engine (Simulation Core) 🦀

**Status:** 🚧 In Active Development

A dedicated physics and logic engine built with **Rust** and compiled to **WebAssembly (WASM)** via `wasm-pack`. This engine powers the **WebGPU-accelerated** _Vibe Heist_ project, handling performant 3D physics calculations off the main JavaScript thread.

## ⚡ Tech Stack

- **Language:** Rust
- **Physics:** Rapier3D
- **Math:** Nalgebra
- **Compilation:** WASM-Bindgen / `wasm-pack`

## 🔧 Build Instructions

**Prerequisites:** Rust, Cargo, `wasm-pack`.

```bash
# Emit the shared layout constants for the client
# (writes ../client/src/generated/sim-layout.ts)
cargo run --bin gen_layout

# Compile to WebAssembly (emits pkg/)
wasm-pack build --target web
```

## 🧪 Tests

The crate also builds natively (`rlib`), so the deterministic replay tests run as plain `cargo test` — no browser or WASM needed:

```bash
cargo test
```

These drive the vehicle through scripted input sequences (drive, brake-to-reverse, handbrake, steering, sleep/wake) and assert on the resulting positions, plus a bitwise determinism check. They are the regression guard for physics changes — run them before pushing.

CI runs `cargo fmt --check`, `cargo clippy --all-targets -- -D warnings`, and `cargo test` on every push.

## 🎯 Architecture

This module is strictly decoupled from the rendering layer. It steps the world at a fixed timestep in a Web Worker, reads inputs from a shared ring buffer, and writes snapshots into a double-buffered `SharedArrayBuffer`, ensuring the simulation remains deterministic and framework-agnostic.

`src/constants.rs` is the **single source of truth** for input bits, SAB layout offsets/strides, and greybox world geometry. The TypeScript mirror (`client/src/generated/sim-layout.ts`) is generated by `cargo run --bin gen_layout` — never edit it by hand; change the constants and regenerate.
